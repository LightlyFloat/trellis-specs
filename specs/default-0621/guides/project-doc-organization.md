# 项目文档组织约定

> 适用范围：所有使用本模板的项目。
> 目的：统一文档存放位置与检索方式，避免文档散落在各目录难以查找。

---

## 一、document/ 目录

在项目根目录创建 `document/` 目录，用于存放**没有固定归属目录的文档**，例如：

- 学习笔记
- 通用参考资料
- 项目文档索引（见下）
- 其他不属于某个具体模块的说明性文档

模块内部的文档（如某模块的部署说明、README）仍保留在该模块目录下，不强制迁入 `document/`。

## 二、文档索引

在 `document/` 目录下维护一份 `文档索引.md`，作为全项目文档的检索入口。要求：

1. 索引头部标注所属项目名称、项目根目录绝对路径、远程仓库链接（若有）。
2. 按用途分类登记，例如：交付文档、规划设计、学习参考、测试记录、汇报材料、历史记录、项目指令等。
3. 每条登记包含：文档名称、相对路径（以项目根为基准）、一句话说明。
4. 新增、移动或删除文档后，同步更新索引。
5. 工具生成文件、临时草稿、第三方依赖文档不纳入索引，并在索引开头注明排除范围。

## 三、维护原则

- 文档索引是“地图”，应始终与实际文件保持一致。
- 新建文档时优先判断归属：属于某模块则放模块目录，无固定归属则放 `document/`。
